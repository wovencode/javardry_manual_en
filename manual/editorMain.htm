Editor [Main]

　In the "Main" tab, you will configure the scenario title and basic settings.

  • Title
    　Enter the scenario name. The title set here will be displayed on the title bar when Javardry is launched.
  • Scenario Identification String
    　Enter a string to identify each scenario. This string is also saved with the save data, and only scenarios with matching strings can load the save data.
  • Title Image
    　Specify the image to be displayed on the title screen, relative to the Javardry "graphic" directory, without the file extension. An image file with "_HR" added to the end of the file name will be recognized as a high-resolution image. Specify the size and position to resize and display the image when the screen size of Javardry is 300*300.
  • Enable Debug Commands
    　Check this option to enable the use of debug commands.
  • [Title Text]
    　This list allows you to set the text to be displayed on the title screen.
    • （Text Area）
      　Enter the text to display. When you click the check button, the actual text with line breaks applied will be displayed for testing. The display is limited to 60 characters (including two half-width characters for forbidden word processing) * 25 lines.
    • left, center, right
      　Specify the alignment of the text: left, center, or right.
    • Display Line
      　Specify which line the text should start displaying from.
  • Music and Sound Effects
    　The names and file names of the music and sound effects used in the scenario. This is a combo box, not a list, and items added via "add" are placed at the very end (at the bottom), and sorting is not allowed. The basic items initially set cannot be renamed or deleted (but the file name can be changed), and duplicate names cannot be added. The file names should be specified without the extension, with music files placed in the "music" directory and sound effects in the "sound" directory.
  • Maximum Party Size
    　Specify the maximum number of party members, from 1 to 6. The default is 6.
  • Front Row Members
    　Specify the number of members in the front row, from 1 to 6. The number of back row members is calculated by subtracting the front row number from the maximum party size. The default is 3.
  • Auto-map Display with M Key
    　When the M key is pressed (it can be changed in Key Settings), you can choose the map display from the following options: "None", "Current Floor Map", "Current Floor Map with Location", "Current Floor Map with Location and Direction", "All Floors Map", "All Floors Map with Location", "All Floors Map with Location and Direction". If auto-mapping is set to none (numeric display) on the title screen, coordinates and direction are shown in "Current Floor Map with Location and Direction" and "All Floors Map with Location and Direction". 　Additionally, if "Trap Display" and "Event Display" are checked, the locations of traps and events are shown on the map. If "Cannot be seen in Dark Zones" is checked, the map will not be displayed even if the M key is pressed in a dark zone (this does not affect map display spells). 　With "Items" and "Flags", conditions can be set for M key map display. If "None" is selected for map display, setting "Items" or "Flags" has no effect. 　"Items" specify that the map will only be displayed if the party possesses the specified items. Items are listed by number and can be specified using commas and hyphen ranges (e.g., 1,5,10-14,20). If no items are specified (when you want the map to display regardless of owned items), leave it empty. 　If "Flags" is checked, the map will only be displayed when the specified special Flag (1) is true. 　If both "Items" and "Flags" are set, the map will be displayed only when both conditions are met. The "Flags" option may not have many uses, but "Items" can be used for map-related items, such as magical maps.
  • Other Settings
    • General
      • Show Help Comments
        　Displays help comments such as recovery values at inns and explanations of spells and items. It also allows viewing the detailed abilities of adventurers.
      • Item Catalog Available
        　If checked, you can view the catalog of identified items. Items that appear in the catalog are those that are not marked as "Do not display in catalog", those acquired in a confirmed state, those identified from an uncertain state, those sold to shops, and items initially owned by the PC. 　The descriptions in the catalog will be shown even if "Show Help Comments" is off.
      • Monster Catalog Available
        　If checked, you can view the catalog of confirmed encounter monsters. 　The descriptions of monsters will be shown even if "Show Help Comments" is off.
      • Do Not Show Modified Attribute Values
        　If checked, even if attribute values are modified by items or effects, the status screen will only show the original unmodified values.
      • Show Modified Attribute Values in Parentheses
        　If checked, modified attribute values will be displayed in parentheses along with the original values, as long as both values fit within two digits.
      • No Re-roll for Bonus Dice
        　If checked, re-rolling for bonus dice when creating a PC in the training hall will not be allowed.
      • Treat Attribute Value of 0 as Unset When Loading Play Data
        　Normally, leave this unchecked.
        　In Javardry Ver.1.045 and Javardry HTML5 Ver.0.164, there was an issue where unset attribute values were saved as 0 instead of -1. If this option is checked, attribute values of 0 will be treated as unset and replaced with the initial values of the race.
    • Facilities
      • Allow Any Alignment to Form Party at Tavern
        　Adventurers of any alignment can freely form parties. If unchecked, adventurers of opposite alignments cannot form parties at the tavern. This can be reversed using a special Flag (0).
      • Do Not Show Sold Out in Shops
        　If checked, items out of stock in shops will not display as "Sold Out", and the shop list will compact without empty spaces.
      • Display Attack Range in Shop List
        　If checked, the attack range of each item will be displayed in the shop list.
    • Spells
      • Retain Spell Effects When Resuming
        　When resuming a paused adventure, previously cast spell effects are retained. If unchecked, spell effects will be cleared.
      • Set Initial Teleport Cursor Position to (0,0) on Map
        　If unchecked, the cursor for teleport spells will initially position at the current location when auto-map is enabled, allowing confirmation of the coordinates. Checking this will set the initial cursor position to coordinates (0,0).
      • Group Spells Affect All PCs
        　If checked, group-targeting spells cast by monsters will affect all PCs in the party. If unchecked, front and back rows will be treated as separate groups.
      • Ensure at Least One Spell is Learned Upon Level Up
        　If a spell acquisition attempt fails during level-up, at least one spell from the available list will be learned.
      • Show Remaining Duration of Active Spells During Dungeon Travel
        　If checked, the remaining duration of active spells affecting the party or individual adventurers during dungeon travel will be displayed. Duration is shown in steps or as "floor" (modifiable in defaultMessage.txt).
      • Do Not Display Effects on Hidden Attributes
        　If checked, the effects of spells modifying hidden attributes will not be shown in battle messages, status displays during combat, or during camp status checks.
    • Item
      • Items are obtained in a confirmed state
        　When obtaining items in a dungeon, they will be obtained in a confirmed state. If unchecked, items will be obtained in an uncertain state.
      • Items are obtained in order starting from the first member
        　When checked, items will be obtained in order starting from the first member of the party. If unchecked, items will be randomly obtained from the members who have empty slots in their inventory. If you want to match the behavior of Wizardry, uncheck this option.
      • No curse on failed identification
        　When failing to identify a cursed item, it will no longer be cursed (although the panic state will still occur).
        　If unchecked, meaning the item is cursed on failed identification, items equipped by other party members that are unidentified cannot be identified.
      • Cursed items do not trigger critical hits or special attacks
        　When cursed, critical hits or other special attacks will not trigger.
      • Cursed items cannot be used
        　Items equipped with curses (in a cursed state) cannot trigger effects or use SP.
      • Items do not disappear when de-cursed at a shop
        　When a cursed item is de-cursed at a shop, it will not disappear but remain.
      • Show equipment requirements
        　When examining or using items in a tavern or during camp in a dungeon, equipment requirements (class and race) will be shown. However, this will only work if the general "Help comment display" option is checked.
    • Dungeon
      • Force use of auto-mapping
        　This will disable the option to choose auto-mapping and will force it to be enabled.
      • Force disable auto-mapping
        　This will disable the option to choose auto-mapping and will force it to be disabled.
      • Hidden door exploration command
        　This will force the hidden door exploration command to appear in the dungeon's sub-menu. Even if unchecked, the command will appear if any hidden doors are present in the dungeon with side story specifications, so it's usually fine to leave it unchecked.
      • Unlock command
        　This will force the unlock command to appear in the dungeon's sub-menu for locked doors. Even if unchecked, the command will appear if any locked doors are present in the dungeon, so it's usually fine to leave it unchecked.
      • Lost in stone
        　When teleporting incorrectly and getting stuck in a stone or floating in the air, the entire party will be lost. If unchecked, living adventurers will die, and their bodies will be transferred to the temple.
      • Do not open camp upon entering dungeon
        　When entering a dungeon, the game will not start from the camp screen but from the dungeon transition screen. However, regardless of this setting, when resuming the adventure, the game will start from the camp screen.
        　This setting can be useful when you don't want to skip over starting point events via teleportation.
      • Return party to the surface on total party wipe
        　If the party is wiped out in a dungeon, normally their bodies will remain on the spot, but if checked, their bodies will be transferred to the surface.
    • Combat
      • Cast spells on preemptive attack
        　You can cast spells when performing a preemptive attack. If the monsters perform the preemptive attack, they will also use spells. If unchecked, spells cannot be cast, but there are no restrictions on using breath attacks or items.
      • Attack the first monster in the group
        　When multiple PCs attack the same monster group with weapons, if checked, they will attack the first monster in the group (following the FC version of Wiz#1). If unchecked, the PCs will attack in order from the first monster in the group (following the FC version of Wiz#2).
        　This can be difficult to visualize, so I'll explain with images.

        　According to "Everything About Wizardry" by Benimatsu, in FC version Wiz#1, PCs attack the first monster in the group, and when checked, this behavior will be followed. If there are multiple monsters in the group and adventurers A to C attack the group, all three adventurers will attack the first monster A in the group.

        　In FC version Wiz#2, the PCs attack in order from the first monster in the group, and when unchecked, this behavior will be followed. If there are multiple monsters in the group and adventurers A to C attack the group, adventurer A will attack monster A, adventurer B will attack monster B, and adventurer C will attack monster C.

        　Now, in the FC version Wiz#2 specification, what happens if there are more adventurers attacking than monsters in the group? In that case, the attacking adventurers will go back to attacking from the first monster. If there are two monsters in the group and three adventurers attack, adventurer A will attack monster A, adventurer B will attack monster B, and adventurer C will attack monster A. If adventurer D also attacks, they will attack monster B.
        　Because of this specification, "Everything About Wizardry" concludes that placing the adventurer with the highest attack power in the second position is optimal in the FC version Wiz#2, but in Javardry, if the monster being attacked is already dead, the attacking adventurer will target the first surviving monster in the group, so this doesn't require much concern.
        　The FC version Wiz#2 specification imagines a front-line warrior keeping each monster in check, but at least in Javardry, the coordination with group attack spells can be beneficial, and it seems more like a romantic setting that might disadvantage adventurers.

      • Change attack target
        　If the target group has been wiped out during a PC's turn, the attack target will change to another group, and the action will continue. If unchecked, no action will occur.
      • Do not interrupt actions upon enemy wipe
        　Even if all enemies are wiped out, the PC's action (spells, item usage, etc.) during that turn will continue.
      • "Hide" available
        　During combat, adventurers with thief abilities will be able to use "Hide." If unchecked, ambushes (surprise attacks) will also be disabled.
      • Do not display healing effect messages
        　Healing effect messages (including persistent healing effects) will be hidden during combat. Check this if you want to conceal the healing abilities of monsters, for example.
      • Do not display modification messages for AC, etc.
        　Modification messages for AC, hit rate, spell nullification rate, spell resistance, breath resistance, spell sealing, breath barrier, stat modifications, additional damage modifications, temporary HP changes, attack count modifications, etc. will be hidden during combat.
      • Directly obtain items after combat victory
        　When victorious in combat, instead of a treasure chest appearing, you will directly obtain items or money. If you don't need to check for traps or disarm them, check this option.
      • No treasure chest if all monsters escape
        　When checked, if all monsters escape (without being defeated) during combat, no treasure chest will appear. Direct item acquisition and money will also be omitted.
      • Weapon switching available
        　When checked, weapons (and shields) can be changed during combat.
